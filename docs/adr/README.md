## ADRとは

Architecture Decision Records (ADR) は、ソフトウェアアーキテクチャに関する重要な決定と、その決定に至った経緯・根拠を記録するドキュメントである。

### フォーマット

各ADRは以下の構成に従う:

- **ステータス**: 提案 / 承認 / 却下 / 廃止 / 検討中
- **コンテキスト**: なぜこの決定が必要か
- **検討した選択肢**: 比較検討した案
- **決定**: 最終的に選んだ方針
- **根拠**: なぜその決定をしたか
- **影響**: この決定がプロジェクトに与える影響
